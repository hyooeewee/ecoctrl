# packages/server

See [root CLAUDE.md](../../CLAUDE.md) for global rules.

Fastify 5 + TypeScript (ESM) backend API. Drizzle ORM + PostgreSQL.

## Constraints

- **Never modify files under `drizzle/`** — auto-generated by `drizzle-kit`.
- Schema change flow: edit `src/schemas/*.ts` → `pnpm db:generate` → `pnpm db:migrate`.
- Register new routes in `src/routes/api.ts`.
- Database connection plugin: `src/plugins/database.ts`.
- Environment variables in `.env.local`.

## Conventions

### Path Aliases

- `@/` → `./src/*`

### Repository Naming (Prisma Client style)

| Operation     | Pattern               | Return      |
| ------------- | --------------------- | ----------- |
| Create        | `createXxx`           | Full object |
| List          | `findManyXxx`         | `T[]`       |
| Find one      | `findXxxByYyy`        | `T \| null` |
| Find or throw | `findXxxByYyyOrThrow` | `T`         |
| Update        | `updateXxx`           | `T \| null` |
| Delete        | `deleteXxx`           | `T \| null` |
| Upsert        | `upsertXxx`           | `T`         |
| Count         | `countXxx`            | `number`    |

- Never return `boolean` or `void` from `create` / `update` / `delete`.
- `findMany` always returns an array; `findBy` returns a single nullable record.

### API Design

- Plural nouns for routes: `/users`, `/reports/plans`.
- Max two nesting levels.
- Methods: `GET` (read), `POST` (create), `PUT` (full replace), `PATCH` (partial update), `DELETE` (delete).
- Success: return resource directly (no `{ data: ... }` wrapper).
- Error shape: `{ error: string }`, with `details` when helpful.
- JWT Bearer token: `Authorization: Bearer <accessToken>`.
- Public path whitelist: `src/routes/api.ts` (`publicPaths`).

### Swagger / OpenAPI

- Every route must declare `tags` for Swagger UI grouping.
- Authenticated routes: `security: [{ bearerAuth: [] }]`.
- Public routes: explicit `security: []`.
- `summary` is required.
- Response schemas via `zod` with `fastify-type-provider-zod`.

### File Upload

- Schema must declare `consumes: ["multipart/form-data"]`.
- Stream to temp file → validate → move to target → persist.
- **Always clean up temp files in `try/catch/finally`.**
- When deleting a resource, use the record returned by `deleteXxx` to locate and remove physical files.

### Error Handling

- Repository layer throws; do not handle HTTP here.
- Route layer catches and maps to appropriate HTTP status codes.
- Always throw `Error` instances; no bare `throw`.
