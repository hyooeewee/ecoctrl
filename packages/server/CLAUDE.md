# packages/server

EcoCtrl backend API service. Fastify 5 + TypeScript (ESM).

## Rules of Engagement

1. **Always output an execution plan before any code change.**
2. The plan must include:
   - List of affected files with full paths.
   - Intended action per file (create / modify / delete).
3. Wait for explicit confirmation ("确认", "可以", or "执行") before proceeding.

## Stack

| Layer | Technology |
|---|---|
| Framework | Fastify 5, TypeScript (ESM) |
| ORM | Drizzle ORM + drizzle-kit |
| Database | PostgreSQL (`postgres` driver) |
| Build | Rolldown (`rolldown.config.ts`) |
| Process Manager | PM2 (`ecosystem.config.cjs`) |
| Dev Runner | tsx (watch mode) |

## Common Commands

```bash
pnpm dev              # dev mode (tsx watch)
pnpm build            # bundle to dist/
pnpm preview          # staging build + run
pnpm start            # PM2 production start
pnpm stop             # PM2 stop
pnpm restart          # PM2 restart
pnpm check            # tsc --noEmit
pnpm clean            # rm dist/

# Database
pnpm db:generate      # drizzle-kit generate
pnpm db:migrate       # drizzle-kit migrate
pnpm db:push          # drizzle-kit push
pnpm db:studio        # Drizzle Studio
pnpm db:seed          # run scripts/seed.ts
pnpm db:reset         # run scripts/reset.ts
pnpm db:drop:db       # run scripts/drop.ts
pnpm db:refresh       # drop + push + seed + studio
pnpm db:drop          # drop last migration
```

## Project Structure

```
src/
  routes/         # Fastify route modules (by domain)
  schemas/        # Drizzle ORM table schemas
  repositories/   # Data access layer (Drizzle queries)
  plugins/        # Fastify plugin registration
  lib/            # Utilities and shared logic
  types/          # Server-specific TypeScript types
  config/         # Configuration files
scripts/          # Database scripts (seed, reset, drop)
drizzle/          # Migration output directory (auto-generated)
```

## Path Aliases

- `@/` → `./src/*` (configured in `tsconfig.json` and `rolldown.config.ts`)

## Critical Constraints

- **Never modify files under `drizzle/`** — auto-generated by `drizzle-kit`.
- Schema change flow: edit `src/schemas/*.ts` → `pnpm db:generate` → `pnpm db:migrate`.
- Register new routes in `src/routes/api.ts`.
- Database connection plugin: `src/plugins/database.ts`.
- Environment variables in `.env.local` (`DATABASE_URL`, `PORT`, `HOST`, etc.).

## Scope

- Modify only files inside `packages/server`.
- `packages/shared`, `apps/*`, and root-level files are **read-only**. Suggest changes to those if needed.

## Repository Naming Convention

Prisma Client style. All repository functions live in `src/repositories/`.

| Operation | Pattern | Return | Notes |
|---|---|---|---|
| Create | `createXxx` | Full object | Never `void` or bare `id` |
| List | `findManyXxx` | `T[]` | Empty array on no results |
| Find one | `findXxxByYyy` | `T \| null` | e.g. `findUserById` |
| Find or throw | `findXxxByYyyOrThrow` | `T` | For hot-path lookups where missing is exceptional |
| Update | `updateXxx` | `T \| null` | Never `boolean` |
| Delete | `deleteXxx` | `T \| null` | Returns deleted record for cleanup |
| Upsert | `upsertXxx` | `T` | Insert or update, return final record |
| Count | `countXxx` | `number` | Add when needed |

**Constraints:**
- No `boolean` or `void` returns from `create` / `update` / `delete`.
- `findMany` always returns an array; `findBy` returns a single nullable record.

## API Design Convention

### Routes & Methods

- Plural nouns: `/users`, `/reports/plans`.
- Max two nesting levels: `/users/:id/avatar`, `/users/:id/preferences`.
- Methods:
  - `GET` — read
  - `POST` — create
  - `PUT` — full replacement
  - `PATCH` — partial update / merge
  - `DELETE` — delete

### Responses

- Success: return the resource object or array directly (no `{ data: ... }` wrapper).
- Created: `201` + full object.
- No content (optional): `204`.
- Error shape: `{ error: string }`, with `details` when helpful.

| Status | Usage |
|---|---|
| `400` | Validation failure, wrong code |
| `401` | Invalid or missing JWT |
| `403` | Insufficient permission (e.g. wrong old password) |
| `404` | Resource not found |
| `409` | Conflict (username/email already taken) |

### Authentication

- JWT Bearer token: `Authorization: Bearer <accessToken>`.
- Refresh token endpoint: `/auth/refresh` with rotation.
- Public path whitelist lives in `src/routes/api.ts` (`publicPaths`).

### Swagger / OpenAPI

- Every route must declare `tags` for Swagger UI grouping.
- Authenticated routes: `security: [{ bearerAuth: [] }]`.
- Public routes: explicit `security: []`.
- `summary` is required.
- Response schemas defined with `zod`; use `fastify-type-provider-zod`.

### File Upload

- Schema must declare `consumes: ["multipart/form-data"]`.
- Flow: stream to temp file → validate → move to target → persist.
- **Always clean up temp files in `try/catch/finally` or equivalent.**
- When deleting a resource, use the record returned by `deleteXxx` to locate and remove physical files.

### Error Handling

- Repository layer throws; do not handle HTTP here.
- Route layer catches and maps to appropriate HTTP status codes.
- Always throw `Error` instances; no bare `throw`.
