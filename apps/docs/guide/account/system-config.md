# 系统全局配置

![系统全局配置](/screenshots/admin-config.png)

系统配置页面（Config）提供平台级别的全局设置。

## 通用设置

| 设置项       | 默认值        | 说明                                               |
| ------------ | ------------- | -------------------------------------------------- |
| 平台名称     | EcoCtrl       | 显示在 Web 门户导航栏和 Admin 后台浏览器的标签标题 |
| 数据刷新间隔 | 30s           | 可选：10s / 30s / 1m / 5m = 管理总览的自动刷新周期 |
| 系统时区     | Asia/Shanghai | 影响日志时间戳和定时任务的执行时间                 |

## 邮件服务

配置 SMTP 用于发送验证码和通知邮件。

| 设置项      | 必填 | 说明                                        |
| ----------- | ---- | ------------------------------------------- |
| SMTP 主机   | 是   | 邮件服务器地址                              |
| SMTP 端口   | 是   | 一般为 465（SSL）或 587（TLS）              |
| 发件人地址  | 是   | 发送邮件的邮箱地址                          |
| 密码/授权码 | 是   | 邮箱密码或 SMTP 授权码（支持切换可见性 👁） |
| SSL/TLS     | 否   | 加密方式开关                                |

### 验证配置

配置完成后，点击「验证 SMTP 配置」：

1. 系统使用当前配置发送测试邮件到管理员邮箱
2. 成功收到测试邮件 → 配置可用
3. 发送失败 → 显示错误信息

> 注意：首次部署时，系统会从 `packages/server/.env.local` 的 `SMTP_*` 环境变量自动同步配置。如需修改请通过 Admin 后台 UI 保存，保存后值写入 `platform_configs` 表，不再依赖环境变量。

## 数据与安全

| 设置项       | 说明                               |
| ------------ | ---------------------------------- |
| 自动备份     | 开关：定时自动备份数据库           |
| 备份保留天数 | 自动备份文件的保留天数             |
| 会话超时时间 | 用户无操作后自动登出的时间（分钟） |

### 自动备份机制

- 基于 `backupSchedule` 配置和 pg-boss 定时任务
- 备份文件存储在服务端磁盘或文件存储中
- 超过保留天数的备份自动清理
