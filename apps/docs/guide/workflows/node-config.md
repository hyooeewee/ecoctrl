# 配置工作流节点

![配置工作流节点](/screenshots/admin-workflow-nodeconfig.png)

## 节点配置面板

点击画布上的节点，右侧弹出 **NodeConfigPanel**。

### 通用配置

所有节点共享的配置：

- **节点名称**：自定义显示名称
- **描述**：节点功能的文字说明

### 节点类型

工作流引擎基于插件架构，节点类型由已安装的插件动态决定。核心节点类型：

#### 控制节点

| 节点      | 功能                    |
| --------- | ----------------------- |
| Start     | 工作流入口，仅一个      |
| End       | 工作流出口，仅一个      |
| Condition | 条件分支（if/else）     |
| Switch    | 多路分支（switch/case） |
| Loop      | 循环执行子流程          |
| Parallel  | 并行执行多个分支        |
| Delay     | 等待指定时间后继续      |

#### 动作节点

| 节点         | 功能                     |
| ------------ | ------------------------ |
| HTTP Request | 发送 HTTP 请求到外部系统 |
| Database     | 执行数据库查询           |
| Email        | 发送邮件                 |
| Variable     | 设置或修改变量           |

### 错误处理

每个节点可配置 **onError** 策略：

| 策略  | 行为                         |
| ----- | ---------------------------- |
| retry | 重试（可配置重试次数和间隔） |
| skip  | 跳过当前节点继续执行         |
| abort | 中止整个工作流               |
| goto  | 跳转到指定节点继续执行       |
