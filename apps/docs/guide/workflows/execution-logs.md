# 执行记录与监控

![执行记录与监控](/screenshots/admin-workflow-executions.png)

工作流的第二个子 Tab 展示最近 50 条执行记录。

## 执行记录列表

| 字段      | 说明                          |
| --------- | ----------------------------- |
| 工作流 ID | 关联的工作流标识              |
| 执行 ID   | 本次执行的唯一标识            |
| 状态      | ✅ 成功 / ❌ 失败 / ⏳ 运行中 |
| 耗时      | 执行总耗时（ms）              |
| 时间      | 执行开始时间                  |

### 实时追加

- 有新的执行开始时，通过 SSE 实时推送到列表顶部
- 无需手动刷新页面
- 正在进行中的执行实时更新状态

### 手动刷新

列表顶部提供「刷新」按钮。

## 执行详情

点击某条记录的「详情」按钮，打开全屏 **LogViewer**：

- 显示工作流执行时的完整节点执行日志
- 每个节点的执行状态（运行中 / 已完成 / 失败 / 跳过）
- 节点的输入参数和输出结果
- 错误堆栈（如有）
- 执行时间线

## 删除记录

- **单条删除**：点击删除按钮 → 确认对话框
- **批量删除**：勾选多条记录 → 点击「批量删除」按钮
- 删除操作不可撤销

## 数据源

执行记录来自 `workflow_executions` 表：

- 执行开始 → 插入记录，状态为 `running`
- 执行完成 → 更新状态和耗时
- 节点日志存储在执行上下文中
