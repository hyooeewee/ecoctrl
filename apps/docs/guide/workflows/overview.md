# 工作流管理概览

![工作流管理概览](/screenshots/admin-workflows.png)

## 工作流列表

以表格形式展示所有已定义的工作流：

| 字段        | 说明                                     |
| ----------- | ---------------------------------------- |
| 名称 + 标签 | 工作流名称和自定义标签                   |
| 触发器类型  | 🔧 手动 / ⏰ 定时 / 🌐 Webhook / 📡 事件 |
| 启用状态    | Switch 开关切换启用/禁用                 |
| 版本        | 当前 DSL 版本号                          |
| 更新时间    | 最后修改时间                             |

### 启用/禁用

- 使用行内 Switch 开关直接切换工作流的启用状态
- 禁用的工作流不会响应触发器，不会被执行

### 手动触发

- 点击行右侧的「触发」按钮立即执行一次工作流
- 适用于测试工作流逻辑
- 执行记录会出现在 [执行记录](/guide/workflows/execution-logs) 中

> 技术细节：工作流引擎架构与 DSL 参考见 [工作流引擎](/reference/subsystems/workflow-engine.md)

## 触发器类型

| 类型     | 说明                                                  |
| -------- | ----------------------------------------------------- |
| manual   | 手动触发，用户点击按钮执行                            |
| schedule | 定时触发，基于 cron 表达式配置                        |
| webhook  | 外部系统通过 HTTP 请求触发（POST /api/webhook/:slug） |
| event    | 系统内部事件触发（如设备告警、能耗阈值超限）          |

## 新建工作流

点击「新建工作流」按钮，创建工作流后进入可视化编辑器。
